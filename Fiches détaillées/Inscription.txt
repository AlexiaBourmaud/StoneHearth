Titre: Inscription à StoneHearth
Acteur: Visiteur
Description:

Pré-requis:


SN: 
    SN1:   Le systeme affiche un bouton inscription sur la page d'accueil 
    SN2:   Le visiteur clique sur le bouton d'inscription
    SN3:   Le systeme affiche une page de formulaire de saisie avec 
             -pseudo
             -mdp
    SN4:   Le visiteur remplit les champs pseudo et mdp
    SN5:   Le systeme verifie si le pseudo est disponible
    SN6:   Le systeme verifie si le mdp est assez sécurisé(il contient plus de 8 caractères)
    SN7:   Le systeme enregistre ces données et authentifie le visiteur
    SN8:   Le systeme affiche "Continuer" ou "remplir coordonnées bancaires"
    SN9:   Le systeme informe que le compte a bien été créee 
    
    
ALT: 
    A1:  Verification pseudo
         En SN5 si le pseudo est non disponible alors afficher "Ton pseudo est déjà pris"
         Retour en SN4
    A2:  Verification mdp
         En SN5 si le mdp est incorrect alors afficher "Ton mdp n'est pas sécurisé"
         Retour en SN4 
    A3:  Sans coordonnées bancaires
         En SN8 si le visiteur clique sur le bouton continue et que les champs sont vides
         Aller en SN12
    A4:  Ajout coordonnées bancaires
         En SN8 si le visiteur décide de remplir ces coordonnées bancaires. Effectuer use case 
         Ajouter coordonnées bancaires
